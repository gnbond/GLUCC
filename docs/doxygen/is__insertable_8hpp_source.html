46/*
47
48On these compilers, `decltype(std::declval<S>() << std::declval<T>())` fails
49(SFINAE does not correctly propagate up to the template specialization so the
50resulting trait gives false positives). It works if the `std::declval<S>()` is
51replaced by either an actual variable (`std::cout` or even `S{}`), but this
52limits the trait to S types that are default-constructable.
53
54As a non-idiomatic work-around, we declare (but do not define) a function to
55return `S&` and use that in `decltype()`. This is more-or-less equivalent to
56`std::declval<S>()` but does not trigger the bug in XCode-10 clang.
57*/
58template <typename S>
59S& S_func();
